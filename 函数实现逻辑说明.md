# 实验一：函数实现逻辑说明

本文档对 `函数任务完成表.md` 中列出的各项函数的具体实现逻辑进行了详细说明。项目由 Rust 编写，使用了与 C++ 不同的特性（如 `HashSet`、`VecDeque` 及枚举等）来实现编译器的底层结构。

## 1. 字符集的创建与运算 (`src/lab1/char_set.rs`)
字符集的核心数据结构是一个全局的静态数组 `P_CHAR_SET_TABLE`，它保存了所有的字符区间段（`CharSet` 结构体）。每个段用 `indexId` 表示所属字符集的编号，`fromChar` 和 `toChar` 表示字符区间的左右边界。通过维护这些线段并确保它们按顺序、不重叠的方式存储，来实现高效的集合运算。

- **`range(from_char, to_char)`**:
  申请一个新的字符集 ID，将指定的字符区间 `[from_char, to_char]` 作为唯一线段直接存入全局静态表中。

- **`union_chars(c1, c2)`**:
  计算两个字符的并集。判断逻辑：
  1. 两字符相同：只需构造单个点段。
  2. 字符在编码上相邻：合并为连续的单线段 `[c1, c2]`。
  3. 字符不相邻：构造两个独立的点段，并归属为同一个新字符集 ID。

- **`union_charset_char(char_set_id, c)`**:
  将单字符并入已知字符集。遍历该字符集的各区间段，判断字符 `c` 是否已包含在线段中、或与某线段左/右邻接。
  若它刚好桥接了两个相连的区间（填补了中间一位的缝隙），则将这两个区间合并为一段；若仅单边相邻，则扩展该段的左/右边界；若完全独立，则增加一个新的单点段。

- **`union_charsets(char_set_id1, char_set_id2)`**:
  两个大字符集合并。提取出两个字符集下的所有区间段，按区间起点 `fromChar` 进行升序排序。然后采用线性扫描法则：判断后一段的起点是否落在前一段的范围内或刚好相邻，从而不断更新重叠区间的 `toChar`，最终把所有融合后的新区间集合存入新的字符集 ID。

- **`difference_charset_char(char_set_id, c)`**:
  从字符集中剔除一个单字符。遍历字符集的所有区间段，判断字符 `c` 落在哪里：
  1. 未命中：当前段整体保留。
  2. 命中某段的左右边界：对该区间的边界作 `-1` 或 `+1` 缩减。
  3. 命中段落中间：将本段在这点一分为二，生成两个新的子区间（即抠掉中间那个字符）。
  4. 命中唯一的单字符点段：直接废弃该段（忽略不写入新集合）。

## 2. NFA 构造法正规运算 (`src/lab1/graph.rs`)
本部分使用 Thompson 构造法，通过定义基础的点和边（状态跳转）拼接生成 NFA。

- **`generateBasicNFA(...)`**:
  构造最基础的正规式 NFA。初始化分配 2 个状态节点（`UNMATCH`(初态) 与 `MATCH`(接受态)），并通过仅有的一条带有 `driverId` 驱动的转移边将它们相连。

- **`union(g1, g2)`**:
  或运算（`g1 | g2`）。新建一个全局的开始节点与结束节点，并引入 $\varepsilon$ 空边（在代码中表示为 `driverId == -1`，`DriverType == "NULL"`）。分别从新头出发连接这两张并行子图 `g1` 和 `g2` 的初始点，再从两图的接受点向新尾节点各连一条空边。所有原有节点将进行编号的整体平移（Shift）以免冲突。

- **`product(g1, g2)`**:
  连接/串联运算（`g1 g2`）。将第一张图 `g1` 的接受结点的状态直接过渡给 `g2` 的初始结点。由于两图串联，直接将前图终点与后图起点融合（或用无条件空边相连），重映射各节点的内部编号即可得到连接后的新图。

- **`plusClosure(g)`**:
  正闭包运算（`g+`）。需要保证子图至少执行一次，同时允许循环。引入新的初态和终态，在从旧终态到新终态加空边的同时，增加一条从 `g` 的旧终态指回 `g` 的旧初态的 $\varepsilon$ 回溯空边。

- **`closure(g)`**:
  克林闭包运算（`g*`）。遵循标准的 Thompson 闭包算法原则：在 `plusClosure` 的基础上，需要处理零次重复的情况。额外添加一条可以直接从新初态直接跳转跃迁到新终态的 $\varepsilon$ 边。

- **`zeroOrOne(g)`**:
  可选运算（`g?`）。同样建立新首尾并产生前后串联空边。为了表达“出现 0 次”，额外构建一条新初始点直接跳到新结束点的空边。此时不需要构建图末尾返回图首的回溯循环边。

## 3. NFA 转化为 DFA (`src/lab1/graph.rs`)
通过经典的子集构造算法（Subset Construction Algorithm），通过闭包扩张和边移动消除不确定性和 $\varepsilon$ 边。

- **`move_by_driver` / `move_by_char`**:
  根据指定字符集（Driver）或具体单字符，去扫描给定的状态集合的转移边。如果当前边集合包含该字符的转移规则，就将指向的 `nextState` 收集起来，产生消耗该输入后一步即可直达的“下一跳节点集”。

- **`epsilon_closure(curr)`**:
  计算指定点集的 $\varepsilon$-闭包。利用广度优先搜索队列（`VecDeque`），从初始点集出发，遍历图中所有的空转移边，收集沿着空边所有能够到达扩列的状态点，直到没有新节点加入为止。

- **`DTran_driver` / `DTran_char`**:
  计算确切的单步状态转换（DFA 边）。这实质是对前两步操作组合封装：首先调用 `move` 计算吃掉输入字符后的一阶变迁点集，然后立即对其做 `epsilon_closure` 求出这些点通过空边可发散到的全量状态稳态集（即 DFA 中的一个状态块）。

- **`NFA_to_DFA(&self)`**:
  子集构造法的核心主控循环。
  首先对 NFA 的 0 号起始节点求 $\varepsilon$ 闭包，将该集合视为 DFA 的绝对初始节点压入工作队列 `queue` 和状态列表 `dfa_state_sets`。然后在循环中逐个弹出状态集，探测在其上应用各种当前可用词法/字符驱动（应用 `DTran` 操作），看会生成哪些新的闭包目标集。若生成的集合是一个从未见过的新目标块，则赋予新状态编号并注册进队列；若已经见过了，则仅增加状态块之间的边关系。循环直至不再产生新的状态组合。

## 4. 最小化 NFA 与后处理优化 (`src/lab1/graph.rs`)
对图中产生的冗余结构（即多余的 $\varepsilon$ 边或者重叠等效状态）进行剪枝、消解操作压缩存储体积。

- **`has_incoming_edge` / `has_outgoing_edge`**:
  在各类图构造时判断特定节点是否存在显式的转移前向边或受控目标边。

- **`closure_conservative(mut g)`**:
  构造期的保守化简。检查在增加克林闭包或类似结构时，可否直接在已有空节点上桥接而无需完全新开辟首尾虚拟空节点，对部分简单的图省略空边的浪费。

- **`build_epsilon_free_nfa(&self)`**:
  全图去 $\varepsilon$ 边化简。通过遍历并预计算每个现存节点向前的全量 $\varepsilon$ 闭包，把带空路径可达的带条件边转化为源节点到目标节点的直接连线边（Bypass）。最后若闭包中包含了 NFA 任何原始终止态，则将自身提权标记同样为可终态。

- **`prune_useless_states(&self)`**:
  孤立节点与死胡同修剪。由终态节点集合开始做图倒序回溯的 BFS：标记出所有能够在反向上溯可达到它的“生产节点 (productive)”。同时在正向上判断“可达节点”。最后丢弃全图中这两项资质有残缺（不可达或执行后注定走向死胡同）的无效节点。

- **`merge_equivalent_states(&self)`**:
  采用状态强等价与聚合思想。遍历所有节点转移的映射关系表，若两个节点拥有对于所有驱动集均指向同一等价目标的转移边集合、并且接收地位一致，认定它们行为强同构，从而“合并全等节点”，并将相关指向旧节点的边统统导向新留存的唯一节点。

- **`minimize_nfa(&self)`**:
  后处理流水线核心调度方法。依序串联进行全套自动降阶压缩：
  1. 调用 `build_epsilon_free_nfa` 去除空边桥。
  2. 调用 `prune_useless_states` 刷除第一波由此暴露出的死分支。
  3. 调用 `merge_equivalent_states` 合并全等闲接节点。
  4. 最终收尾再度作一次死分支裁剪确保纯净最简。
